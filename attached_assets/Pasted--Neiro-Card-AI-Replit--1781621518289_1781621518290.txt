### 3. Балансы карточек в профиле пользователя

В таблицу `profiles` (или `users`) добавь поля:
- `nano2_balance` (INT, default 0)
- `pro_balance` (INT, default 0)

При успешной оплате пакета увеличивай соответствующий баланс на количество карточек.

В редакторе (`/editor`) показывай текущий баланс выбранной модели. Если баланс = 0:
- Для **авторизованных** пользователей блокируй кнопку генерации, показывай сообщение «Недостаточно карточек. Купите пакет» и кнопку «Купить пакет».
- Для **неавторизованных** действует лимит 5 пробных карточек (с водяным знаком). Если лимит исчерпан — предложение войти или зарегистрироваться.

При успешной генерации карточки (только для авторизованных с положительным балансом) уменьшай баланс выбранной модели на 1. Генерация без водяного знака.

### 4. Опция «Создать карточку без надписей»

В левой панели редактора добавь чекбокс «Только фон (без текста)». По умолчанию выключен.

Если включён:
- При генерации карточки создавай только товар + фон (удаление фона, генерация фона), без текстовых оверлеев.
- Списание карточек из баланса происходит как обычно.
- После генерации пользователь может вручную добавить текст, плашки, наклейки.

### 7. Интеграция с ЮMoney (реальные платежи) — НОВЫЙ БЛОК

**Замени заглушки «Оплатить» на реальную интеграцию с ЮMoney через официальный SDK.**

#### Техническая реализация:

**Бэкенд (Express.js):**

1. Установи SDK: `npm install yoomoney-sdk`[reference:2][reference:3]
2. Создай API-маршруты:
   - `POST /api/payment/create` — создаёт платёж и возвращает ссылку на оплату.
   - `POST /api/payment/webhook` — обрабатывает уведомления от ЮMoney об успешной оплате.

3. Логика создания платежа:
   - Получаешь от клиента: `packageId` (идентификатор пакета), `modelType` ('nano2' или 'pro').
   - Находишь цену и количество карточек для этого пакета.
   - Создаёшь платёж с помощью SDK (метод `request-payment` или через API форм оплаты)[reference:4].
   - Возвращаешь клиенту `confirmationUrl` — ссылку, на которую нужно перенаправить пользователя для оплаты.[reference:5]

4. Логика обработки вебхука (уведомления об оплате):
   - ЮMoney присылает POST-запрос на `/api/payment/webhook` после успешной оплаты[reference:6].
   - Проверяешь подпись уведомления (для безопасности)[reference:7].
   - Находишь пользователя по `userId` (переданному в метаданных платежа).
   - Увеличиваешь соответствующий баланс (`nano2_balance` или `pro_balance`) на количество карточек.
   - Отмечаешь платёж как выполненный в БД (таблица `payments`).

**Фронтенд (React):**

1. При нажатии кнопки «Оплатить» на странице пакетов:
   - Отправляешь запрос на `/api/payment/create`.
   - Получаешь `confirmationUrl`.
   - Перенаправляешь пользователя на эту ссылку (через `window.location.href`).

2. После возвращения пользователя на сайт (с редиректа от ЮMoney):
   - Показываешь тост об успешной оплате.
   - Обновляешь баланс пользователя (запросить `/api/user/balance`).

#### Переменные окружения (Secrets):

Добавь в Replit Secrets:
- `YOOMONEY_TOKEN` — твой персональный токен доступа к API ЮMoney[reference:8]. Получить его можно через скрипт из SDK или в личном кабинете ЮMoney[reference:9].
- `YOOMONEY_NOTIFICATION_SECRET` (опционально) — секрет для проверки вебхуков.

#### Документация:
- Официальная документация API ЮMoney: https://yoomoney.ru/docs/wallet[reference:10][reference:11]
- Примеры использования SDK: https://github.com/AlexXanderGrib/yoomoney-sdk/blob/main/examples/README.md[reference:12]

**Важно:** Для тестирования используй песочницу ЮMoney (sandbox). В ней можно совершать тестовые платежи без реальных денег.

### 8. Исправление водяного знака (защита от обхода)

Водяной знак должен быть неотъемлемой частью отображаемого изображения для всех, у кого нет платных карточек.

- Для **неавторизованных** пользователей и для авторизованных с нулевым балансом: водяной знак накладывается на изображение в момент отображения в редакторе (на холсте или в preview).
- Для **авторизованных с положительным балансом**: водяной знак не добавляется.
- Добавь обработчик `contextmenu` на изображение (или на весь компонент предпросмотра), который блокирует стандартное контекстное меню для защищённых изображений: `e.preventDefault()`.
- Создай вспомогательную функцию `applyWatermark(imageUrl, callback)`, которая загружает изображение в canvas, рисует поверх полупрозрачный текст «Neiro Card AI» по диагонали (альфа 0.4, белый цвет) и возвращает dataURL.
