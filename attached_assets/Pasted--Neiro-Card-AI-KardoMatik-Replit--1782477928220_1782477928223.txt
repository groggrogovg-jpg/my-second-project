### 1. Защита от обхода водяного знака через контекстное меню

**Проблема:** При пробном периоде (неавторизованный пользователь или с нулевым балансом) можно сохранить картинку через правую кнопку мыши → «Сохранить картинку как», и она сохраняется без водяного знака.

**Требуемое поведение:**
- Водяной знак должен быть неотъемлемой частью самого изображения, а не просто оверлеем на холсте.
- Для пробных карточек водяной знак накладывается на изображение ДО того, как оно отображается в браузере.
- Реализуй это одним из способов:
  - **Серверный:** при сохранении карточки на сервере (Node.js) накладывай водяной знак с помощью библиотеки `sharp` и сохраняй уже с водяным знаком. Для платных пользователей сохраняй оригинал.
  - **Клиентский (запасной):** при отображении изображения загружай его в canvas, рисуй водяной знак и используй `canvas.toDataURL()` для показа. Заблокируй контекстное меню на canvas: `onContextMenu={(e) => e.preventDefault()}`.
- Убедись, что для авторизованных пользователей с положительным балансом водяной знак НЕ появляется.

**Где менять:** Компонент отображения карточки (скорее всего `Preview.jsx` или `Editor.jsx`) и серверный маршрут сохранения карточки.

### 2. Исправление текста «ВАШ ТОВАР» → «МОДЕЛЬ»

**Где:** На странице редактора (`/editor`), в блоке выбора модели генерации (Nano Banana 2 / Pro).

**Что сделать:** Найти текст «ВАШ ТОВАР» (вероятно, это заголовок или подпись к полю выбора модели) и заменить его на «МОДЕЛЬ». Проверить все файлы, где может встречаться этот текст (обычно в компонентах левой панели или в переводе). Исправить только визуальный текст, не затрагивая логику.

### 3. Восстановление вкладки редактирования карточки после генерации

**Проблема:** Раньше после генерации карточки появлялась вкладка/панель для редактирования (изменение текста, позиций, стилей). Сейчас она пропала.

**Требуемое поведение:**
- После успешной генерации карточки (кнопка «✨ Создать карточку» или AI-генератор) пользователь должен видеть панель редактирования, где можно:
  - Перемещать текстовые блоки (название, цена, преимущества).
  - Изменять текст, цвет, размер шрифта, форму плашки.
  - Добавлять/удалять блоки.
- Проверь, что вкладка «Редактировать» (или соответствующая панель) не скрыта условно. Возможно, она отображается только после сохранения карточки, а не после генерации. Настрой логику так, чтобы она появлялась сразу после генерации (или после того, как пользователь нажмёт «Применить» изменения).
- Убедись, что панель редактирования видна и для пробных, и для платных пользователей.

**Где менять:** Компонент редактора (скорее всего `Editor.jsx` или `LeftPanel.jsx`), логика переключения режимов.

### 4. Админ-панель: добавление восстановления доступа (логин/пароль)

**Задача:** Добавить в админ-панель (`/admin`) возможность восстановления доступа для администратора, если он забыл пароль.

**Что сделать:**
- Создай на странице входа в админку (`/admin/login`) ссылку «Забыли пароль?».
- При клике открывается форма, где нужно ввести email администратора.
- После отправки система генерирует одноразовый токен, сохраняет его в БД (в таблицу `password_resets` с полями `email`, `token`, `expires_at`), и отправляет письмо на email со ссылкой для сброса пароля (например, `/admin/reset-password?token=...`).
- Страница сброса: форма с новым паролем и подтверждением. При успешной отправке обновляется пароль администратора в БД, токен удаляется.
- Если SMTP не настроен, можно использовать заглушку (логирование токена в консоль или создание одноразового пароля, который отображается в админке для разработчика).

**Где менять:**
- Создать новые файлы: `routes/adminAuth.js`, `pages/AdminForgotPassword.jsx`, `pages/AdminResetPassword.jsx`.
- Добавить маршруты на бэкенде: `POST /api/admin/forgot-password`, `POST /api/admin/reset-password`.
- Добавить таблицу в БД (если используется SQLite/PostgreSQL): `CREATE TABLE password_resets (id INTEGER PRIMARY KEY AUTOINCREMENT, email TEXT NOT NULL, token TEXT NOT NULL UNIQUE, expires_at DATETIME NOT NULL);`
- Если используется Replit DB (ключ-значение), создать отдельный ключ для хранения токенов.

**Важно:** Пока что для простоты можно сделать заглушку, которая выводит ссылку для сброса в консоль сервера (или в админ-панель для разработчика), чтобы можно было протестировать, не настраивая почту.
