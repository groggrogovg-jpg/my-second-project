### 1. Промокод для разработчика (пополнение баланса)

- В личном кабинете (`/profile`) для пользователей с `is_developer = true` добавь блок «Промокод разработчика».
- Блок содержит поле ввода текста и кнопку «Активировать».
- При вводе кода `DEV100` (или любого другого, который ты задашь в переменной окружения `DEV_PROMO_CODE`) и нажатии кнопки:
  - Баланс `nano2_balance` увеличивается на 100.
  - Баланс `pro_balance` увеличивается на 100.
  - Показывается тост «Баланс пополнен на 100 карточек Nano2 и 100 карточек Pro».
- Код должен храниться в переменной окружения `DEV_PROMO_CODE` (в Replit Secrets), чтобы можно было менять без перекодировки.
- Если пользователь не `is_developer`, блок не показывать.

### 2. Админ-панель — расширение

Дополни существующую страницу `/admin` (если её нет — создай, доступна только пользователям с `is_developer = true`). Добавь следующие разделы:

#### 2.1. Список пользователей
- Таблица со столбцами:
  - Email
  - Дата регистрации
  - Баланс Nano2
  - Баланс Pro
  - Количество созданных карточек (общее)
  - Статус (активен / заблокирован) — пока заглушка
- Для каждого пользователя добавить кнопки:
  - «Пополнить Nano2 +100» — увеличивает `nano2_balance` на 100
  - «Пополнить Pro +100» — увеличивает `pro_balance` на 100
  - «Сбросить баланс» — устанавливает оба баланса в 0
- Действия должны выполняться через API-запросы с подтверждением (модальное окно).

#### 2.2. История платежей
- Таблица со столбцами:
  - Пользователь (email)
  - Дата платежа
  - Сумма
  - Пакет (модель + количество)
  - Статус (успешно / ошибка)
- Данные брать из таблицы `payments` (если её нет — создать с полями: id, user_id, amount, package_type, status, created_at).

#### 2.3. Логи ошибок генерации
- Таблица со столбцами:
  - Пользователь (email)
  - Модель (Nano2 / Pro)
  - Дата и время ошибки
  - Текст ошибки (из лога)
- Для этого создать таблицу `generation_logs` (если нет) с полями: id, user_id, model_type, status (success/error), error_message, created_at.
- При каждой неудачной генерации (ошибка API, таймаут и т.д.) записывать запись в эту таблицу.
- В админ-панели отображать только ошибки (status = 'error').

### 3. API-маршруты для админ-панели

Создай следующие эндпоинты (только для авторизованных с `is_developer = true`):

- `GET /api/admin/users` — возвращает список всех пользователей с их балансами и датой регистрации.
- `POST /api/admin/users/:userId/balance` — принимает `{ nano2_delta, pro_delta }` и изменяет баланс пользователя на указанные значения.
- `GET /api/admin/payments` — возвращает все платежи с данными пользователя.
- `GET /api/admin/logs` — возвращает логи ошибок с данными пользователя.

Все маршруты должны проверять, что текущий пользователь — разработчик (is_developer = true), иначе возвращать 403.
