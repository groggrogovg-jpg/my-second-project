Ты работаешь над существующим проектом Neiro Card AI в Replit. Не удаляй и не переписывай существующий код — только **добавь** функции аутентификации, перечисленные ниже.

### 1. Восстановление пароля через почту (Replit Auth)

- Включи встроенную функцию Replit Auth для сброса пароля ("Forgot Password").
- На странице входа (`/login`) добавь ссылку «Забыли пароль?».
- При клике открывается форма для ввода email. После отправки Replit Auth автоматически отправляет письмо со ссылкой для сброса.
- Письмо приходит от системы Replit (не нужно настраивать SMTP).
- Добавь сообщение-подсказку: «Если письмо не пришло, проверьте папку "Спам"».

### 2. Вход через Google (Replit Auth)

- Включи OAuth-провайдера Google в настройках Replit Auth (это делается через интерфейс Replit, но ты, Agent, должен подготовить код для обработки callback).
- На странице входа добавь кнопку «Войти через Google» (используй компонент `AuthButton` или аналогичный из Replit Auth).
- Для работы потребуется создать OAuth-приложение в Google Cloud Console и добавить Client ID и Client Secret в Secrets (переменные окружения). В коде проверь наличие этих переменных и, если их нет, покажи сообщение «Настройка Google OAuth не завершена».

### 3. Вход через Яндекс ID (через Clerk Auth, т.к. Replit Auth не поддерживает Яндекс)

- Установи библиотеку `@clerk/clerk-react` и `@clerk/express` (или `@clerk/backend`).
- Настрой Clerk с поддержкой OAuth Яндекс:
  - Создай приложение в Яндекс OAuth (https://oauth.yandex.ru/client/new).
  - Получи Client ID и Client Secret.
  - В Clerk Dashboard добавь Яндекс как Social Provider, укажи ключи.
- На странице входа добавь кнопку «Войти через Яндекс ID».
- При клике пользователь перенаправляется на страницу авторизации Яндекса, после подтверждения возвращается на сайт.
- Clerk автоматически создаёт или находит пользователя по email, полученному от Яндекса.
- Связь с существующими пользователями по email: если пользователь с таким email уже зарегистрирован через почту, Clerk должен связать аккаунты (это настраивается в Clerk Dashboard).

### 4. Общие требования

- Все кнопки входа (почта, Google, Яндекс) должны быть стилизованы в едином дизайне (светлая тема, Tailwind CSS).
- После успешного входа пользователь перенаправляется на `/editor` (или на текущую страницу, если она защищена).
- Добавь обработку ошибок (например, если OAuth-провайдер вернул ошибку — показывать понятное сообщение).

### 5. Переменные окружения (Secrets)

Добавь в Replit Secrets (инструкции для разработчика, который получит ключи):

- `GOOGLE_CLIENT_ID` и `GOOGLE_CLIENT_SECRET` – для Google OAuth.
- `YANDEX_CLIENT_ID` и `YANDEX_CLIENT_SECRET` – для Яндекс OAuth.
- `CLERK_SECRET_KEY` – ключ от Clerk (для Яндекс и других провайдеров).

В коде сделай проверку: если ключи отсутствуют, показывать заглушку «Настройка входа через ... временно недоступна» (но лучше не блокировать основной вход по почте).

### 6. Инструкция для разработчика (вывести после выполнения)

После завершения изменений выведи список:
- Какие файлы добавлены/изменены.
- Где в интерфейсе появились кнопки «Войти через Google» и «Войти через Яндекс ID».
- Какие переменные окружения нужно добавить и как получить ключи для Google и Яндекс OAuth.

**Важно:** Не удаляй существующую логику регистрации по email/паролю. Она остаётся основным способом входа.
